# Plan 16-02: Fix Analyst Agent Encoding + Reorder AI Provider Chain

## Context

The Analyst Agent's OpenRouter integration crashes with a ByteString encoding error because the HTTP header `X-Title: 'Araştır AI'` contains a non-ASCII Turkish character `ş` (U+015F = 351). Additionally, Groq has proven more reliable in production — it should be promoted to primary.

## Tasks

### Task 1: Fix OpenRouter ASCII Header Encoding
**File:** `src/agents/analyst-agent.ts`
**Changes at line 105-108:**

Replace:
```typescript
defaultHeaders: {
  'HTTP-Referer': 'https://arastir.ai',
  'X-Title': 'Araştır AI',
},
```

With:
```typescript
defaultHeaders: {
  'HTTP-Referer': 'https://arastir.ai',
  'X-Title': 'Arastir AI',
},
```

**Rationale:** HTTP headers must be ASCII-safe. Remove the cedilla from `ş` → `s`.

**Verification:** OpenRouter call no longer throws `ByteString` error.

### Task 3: Add JSON Parse Safety for All Providers
**File:** `src/agents/analyst-agent.ts`
**Changes:** Wrap every `JSON.parse(content)` in a safety check that strips markdown code fences before parsing:

```typescript
function safeParseJSON(raw: string): AnalystInsights {
  // Strip markdown code fences if present (```json ... ```)
  let cleaned = raw.trim();
  if (cleaned.startsWith('```')) {
    cleaned = cleaned.replace(/^```(?:json)?\s*/, '').replace(/\s*```$/, '');
  }
  return JSON.parse(cleaned) as AnalystInsights;
}
```

Apply `safeParseJSON()` in all 4 provider blocks instead of raw `JSON.parse()`.

**Rationale:** Some LLM providers occasionally wrap JSON in markdown code fences despite `response_format: json_object`. This prevents spurious parse failures.

**Verification:** AI synthesis succeeds even if LLM returns ```json wrapped output.
