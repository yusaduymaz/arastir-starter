# Plan 16-01: Fix Yahoo Finance v3 + Market Agent Provider Chain

**Phase:** 16 - Multi-Source Research Agents & Live Data Cards
**Goal:** Fix the Yahoo Finance v2→v3 breaking change across all consumers and harden the market data fallback chain.
**Requirements:** AGENT-RAPID-01
**Estimated Tasks:** 5

---

## Context

Yahoo Finance v3.14.1 requires `new YahooFinance()` instantiation before calling any method. The codebase still uses the v2 pattern (direct default export as instance). This breaks:
- Dashboard ticker marquee (ticker-data.ts)
- Research pipeline market data (yahoo-client.ts)
- Alpha Vantage fallback (client.ts) — crashes on undefined `Global Quote`

## Tasks

### Task 1: Create Yahoo Finance v3 Singleton Module
**File:** `src/lib/market/yahoo-instance.ts` (NEW)
**Action:** Create a shared singleton that both `yahoo-client.ts` and `ticker-data.ts` import.

```typescript
// src/lib/market/yahoo-instance.ts
import YahooFinance from 'yahoo-finance2';
const yf = new YahooFinance();
export default yf;
```

**Verification:** File exists and exports a YahooFinance instance.

### Task 4: Add Null Guard to Alpha Vantage Fallback
**File:** `src/lib/market/client.ts`
**Changes at line 72-85:**
- Add null check for `obj['Global Quote']` before accessing nested fields
- If `Global Quote` is missing, throw descriptive error instead of crashing on undefined

```typescript
const raw = obj['Global Quote'];
if (!raw || typeof raw !== 'object') {
  throw new Error(`Alpha Vantage returned no quote data for ${symbol}`);
}
```

**Verification:** When Alpha Vantage returns empty/unexpected response, error is caught by `getFullMarketData` and logged cleanly (no `Cannot read properties of undefined`).

If v3 singleton causes issues, revert to pinning `yahoo-finance2@2.x` in `package.json`. However, v3 is the supported version — singleton pattern is the correct fix.
