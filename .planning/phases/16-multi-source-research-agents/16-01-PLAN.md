# Plan: 16-01 - Integrate RapidAPI Providers into Market & News Agents

**Phase**: 16 - Multi-Source Research Agents & Live Data Cards
**Goal**: Integrate the newly built RapidAPI providers into the existing `market-agent` and `news-agent`.

## Rationale
The core goal of Phase 16 is to surface the gathered external market data in the user experience. By updating our agents to use these new providers, we improve data depth and resilience.

## Key Deliverables
1. **Market Agent Upgrade**: Update `market-agent` to merge data from YH Finance, Yahoo Real-Time, and Real-time Finance Data.
2. **News Agent Upgrade**: Update `news-agent` to ingest CNBC, Crypto News, and Turkey News Live data.
3. **Precedence Logic**: Implement a defined precedence for source data to avoid collisions.
4. **Provenance Metadata**: Ensure every data object in the agent output includes the `source` provenance tag.

## Implementation Steps
### 1. Market Agent Integration
- **File**: `src/agents/market-agent.ts`
- **Actions**:
    - Update the agent's fetching logic to call the new provider clients.
    - Implement merging logic: use the most relevant provider based on the user's ticker.
    - Ensure output schema remains compliant with the `research_sessions` table structure.

### 2. News Agent Integration
- **File**: `src/agents/news-agent.ts`
- **Actions**:
    - Update the agent's fetch logic to include CNBC, Crypto News, and Turkey News.
    - Implement a filtering layer to ensure news results are relevant to the queried ticker.

### 3. Verification
- **Actions**:
    - Run existing unit tests for these agents to ensure no regressions.
    - Execute a test research request and inspect the `research/` JSON output for the provenance tag.
