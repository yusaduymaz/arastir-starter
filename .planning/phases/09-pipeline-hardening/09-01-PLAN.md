<objective>
Eliminate the ESM/CJS mismatch that prevents data agents from running, standardize file paths across orchestrator and agents, and surface missing env vars at startup so degraded modes are obvious to operators.
</objective>

<context>
This plan is the diagnostic-grade fix for the dashboard error "Ajanlar başarılı görünüyor ancak araştırma çıktı dosyaları bulunamadı." Root cause: `package.json` declares `"type": "module"` but agents were spawned via `ts-node --project tsconfig.scripts.json`, which fails to resolve extensionless TS imports under ESM. Every agent crashed with `ERR_MODULE_NOT_FOUND` before writing any file — orchestrator then threw the generic "files not found" error. Hotfix applied (tsx + ESM-safe entry guards + process.cwd() paths) in pre-Phase-9 work; this plan formalizes and extends it.
</context>
