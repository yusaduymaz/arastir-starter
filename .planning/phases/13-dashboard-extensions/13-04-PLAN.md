  <action>
  Yeni dosya olustur: src/lib/market/ticker-data.ts

  1. TickerQuote interface tanimla -- MarqueeTicker'in beklentisiyle uyumlu:
     - symbol: string (gosterim adi: THYAO, DOLAR, EURO, ALTIN vb.)
     - price: string (formatlanmis fiyat)
     - change: string (ornek: "+2.34%")
     - changePercent: number (ham sayi -- TRENDING siralamasi icin)
     - up: boolean

  2. Sembol sabitleri tanimla:
     - BIST_SYMBOLS: 13 eleman -- THYAO.IS, EREGL.IS, SASA.IS, ASELS.IS, GARAN.IS, AKBNK.IS, KCHOL.IS, TUPRS.IS, BIMAS.IS, FROTO.IS, PGSUS.IS, TOASO.IS, VESTL.IS
     - FX_COMMODITY_SYMBOLS: TRY=X (USD/TRY), EURTRY=X (EUR/TRY), GLDTR.IS (Istanbul Gold ETF -- TRY bazli)
     - ALL_SYMBOLS = spread her ikisini birlestir

  3. In-memory cache tanimla -- client.ts'deki Map pattern'ini takip et:
     - CACHE_TTL_MS = 10 * 60 * 1000 (10 dakika)
     - tickerCache: Map<string, { data: TickerQuote[]; ts: number }>
     - Cache key olarak sabit 'dashboard-tickers' string kullan

  4. fetchTickerData() async fonksiyonu (internal, export etme):
     - YahooFinance instance olustur (new YahooFinance())
     - quote(ALL_SYMBOLS, {}, { validateResult: false }) ile batch cagri yap
     - Yaniti any[] olarak al
     - Her sonucu filtrele: q && q.regularMarketPrice != null olanlari al
     - Her sonucu TickerQuote'a map'le:
       a. symbol: .IS suffix'ini strip et, sonra display mapping uygula
       b. Display mapping: TRY=X -> DOLAR, EURTRY=X -> EURO, GLDTR.IS'den .IS strip edince GLDTR -> ALTIN
       c. price: Intl.NumberFormat('tr-TR') ile formatla. Fiyat >= 10000 ise minimumFractionDigits=0, aksi halde minimumFractionDigits=2 kullan
       d. change: regularMarketChangePercent'i al, >= 0 ise "+" prefix ekle, .toFixed(2) + "%" formatla
       e. changePercent: regularMarketChangePercent'in ham number degeri
       f. up: regularMarketChange >= 0
     - try/catch ile sar. Hata durumunda console.warn('[Ticker] Batch fetch failed:', err.message) logla ve bos array don
     - Bireysel sembol hatalari zaten filtre adiminda elenecek (null regularMarketPrice)

  5. getTickerData() export fonksiyonu:
     - Cache hit kontrolu yap (tickerCache.get('dashboard-tickers'))
     - Gecerli cache varsa (Date.now() - cached.ts < CACHE_TTL_MS) cache'i don, console.log ile cache hit logla
     - Cache miss ise fetchTickerData() cagir
     - Sonuc bos degilse cache'e yaz
     - Sonucu don

  ONEMLI: unstable_cache KULLANMA. Proje genelinde in-memory Map cache pattern'i tercih ediliyor (client.ts'deki gibi). unstable_cache Next.js Data Cache'e bagimli ve force-dynamic sayfada gereksiz karmasiklik ekler.
