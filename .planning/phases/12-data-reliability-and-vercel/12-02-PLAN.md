# Phase 12: Data Reliability & Vercel Compatibility
## Plan 12-02: Fix News Search Relevance & Sentiment

### Objective
Ensure the News Agent returns highly relevant news articles for the specific ticker requested, and improve the resilience of the news extraction process.

### Context & Rationale
During earlier testing, the News Agent sometimes fetched generic market news or failed to accurately filter articles specific to the requested company (e.g., fetching general BIST100 news when asked for "THYAO"). A robust financial research tool must strictly filter noise. Additionally, we need to ensure the sentiment analysis pipeline is stable if an LLM API key is provided.

### Scope
- Refactor `src/lib/news/api_client.ts` (or the respective search implementation) to use strict query parameters or advanced search syntax (e.g., exact match quotes) when searching for the ticker or company name.
- Implement a secondary relevance check in the News Agent: discard articles where the ticker or company name does not appear in the title or first paragraph.
- Ensure the agent handles rate limits or search API failures gracefully by falling back to alternative sources if necessary.

### Implementation Steps
1. **Enhance Search Logic**: Modify the news search query to include both the ticker (e.g., "THYAO") and the long-form company name (e.g., "Türk Hava Yolları") for better coverage and relevance.
2. **Relevance Filter**: Add a post-processing step in `src/agents/news-agent.ts` that filters out articles that don't mention the target entities.
3. **Sentiment Integration**: Ensure the News agent assigns a basic sentiment (positive, neutral, negative) to the retained articles, either heuristically or via a lightweight LLM call if configured.

### Verification (Goal-Backward)
- [ ] When searching for a specific ticker, do all returned news articles explicitly mention the company?
- [ ] Are generic market news articles successfully filtered out?
- [ ] Is the data properly saved to the `agent_runs` table?
