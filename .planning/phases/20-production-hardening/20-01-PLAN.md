# Plan 20-01: Arka Plan İş Kuyruğu (Upstash QStash) Entegrasyonu

## Objective
Araştırma pipeline'ını (`executeResearchPipeline`) API route'undan ayırıp asenkron bir web hook / background job yapısına (Upstash QStash) geçirerek Vercel serverless fonksiyon limitlerinden (60sn/300sn) kaynaklı kesintileri ortadan kaldırmak.

## Context & Rationale
Vercel gibi ortamlarda API route'ları hızlı yanıt vermelidir. Uzun süren (agent + scraping + LLM) işlemleri `fire-and-forget` (await etmeden) şeklinde başlatmak, API yanıtı döndükten sonra Vercel'in execution context'ini sonlandırmasıyla işlemin yarıda kesilmesine sebep olur. Bu durum production için kabul edilemez. Upstash QStash ile `/api/research` rotası işlemi sadece kuyruğa atacak ve dönecektir.

## Implementation Steps

1. **QStash Client & API Rota Hazırlığı:**
   - `@upstash/qstash` paketini yükleyin.
   - `/api/qstash/research` rotasını (veya benzeri bir webhook rotası) oluşturun. Bu rota `verifySignature` ile sadece QStash'ten gelen istekleri kabul edecek şekilde korunmalıdır.

2. **`executeResearchPipeline` Taşınması:**
   - `src/app/api/research/route.ts` içindeki ağır asenkron mantığı (pipeline'ın kendisi), yeni QStash endpoint'ine (`/api/qstash/research/route.ts`) taşıyın.

3. **Tetikleyiciyi Güncelleme:**
   - `/api/research/route.ts` içinde artık işlemi çağırmak yerine, oluşturulan session ID ve ticker bilgisiyle QStash'e (veya Trigger.dev/Inngest) bir mesaj yayınlayın (`publish`).

4. **Retry ve Hata Yönetimi:**
   - QStash endpoint'i başarısız olursa (`catch` bloğu), `research_sessions` tablosunu `status='failed'` olarak güncelleyin ki frontend'de sonsuz yükleme durumu olmasın.

## Execution
- `executeResearchPipeline` QStash handler'ına taşındı.
- Mevcut `POST /api/research` işlemi sadece veritabanı kaydını yaratıyor ve kuyruğa mesaj atıyor.

## Verification
- Sisteme `/api/research` üzerinden bir hisse sorgusu atın.
- Vercel veya lokal dev ortamında isteğin hemen 202 döndüğünden emin olun.
- Arka planda webhook'un tetiklendiğini, QStash dashboard'unda başarılı bir deliver işlemi göründüğünü ve Supabase'deki kaydın `completed` olarak güncellendiğini doğrulayın.
