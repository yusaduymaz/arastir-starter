# Plan 20-03: Upstash Redis Rate Limiting

## Objective
Araştırma endpoint'lerine (ve kritik diğer API endpointlerine) IP veya User ID bazlı hız sınırlandırması (Rate Limit) getirerek sistemi kötü niyetli spam kullanıcılara (DDoS veya token exhaust saldırılarına) karşı korumak.

## Context & Rationale
Token bittiğinde istek 403 alsa bile bu işlem veritabanını meşgul eder. Bir botun dakikada 5000 istek atması Supabase veritabanını tüketebilir. Edge katmanında veya route handler'ın en üstünde hafif, hızlı bir Redis limitörüne ihtiyaç var. 

## Implementation Steps

1. **Paket ve Upstash Hazırlığı:**
   - `@upstash/ratelimit` ve `@upstash/redis` paketlerini kurun.
   - Projenin `.env.local` ve Vercel env'larına `UPSTASH_REDIS_REST_URL` ile `UPSTASH_REDIS_REST_TOKEN` ekleyin.

2. **Rate Limit Client Oluşturma:**
   - `src/lib/ratelimit.ts` dosyası açıp, Redis tabanlı (örneğin dakikada 5 istek sınırı olacak şekilde) bir ratelimit instance'ı (`SlidingWindow` vs) oluşturun.

3. **API Rotasında Uygulama:**
   - `src/app/api/research/route.ts` en tepesine bu kontrolü ekleyin.
   - Request'i yapan User ID veya IP adresi üzerinden (`req.headers.get('x-forwarded-for')`) limit kontrolü yapın.
   - Eğer limit aşıldıysa anında `429 Too Many Requests` HTTP kodunu dönün.

## Execution
- `@upstash/ratelimit` entegre edildi ve `/api/research` endpointine eklendi.

## Verification
- Aynı kullanıcı / IP ile kısa sürede 6 istek atın.
- 6. isteğin 429 status code döndüğünü ve işlemin hiç Supabase'e ulaşmadığını doğrulayın.
