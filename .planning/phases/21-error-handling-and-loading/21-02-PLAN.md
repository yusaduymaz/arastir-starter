# Phase 21: Error Handling & Loading States - Plan 2: Page Transition Loading States & Skeletons

## Task 1: Create Dashboard Loading State
- Create `src/app/dashboard/loading.tsx`.
- Implement a visually appealing loading screen that fits the dark terminal UI (e.g., a centered animated spinner, a pulsing logo, or a skeleton layout that mimics the dashboard structure).
- This will be shown when navigating from the landing page to the dashboard.

## Task 2: Create Root Loading State
- Create `src/app/loading.tsx` as a fallback loading state for the rest of the application (e.g., between the landing page and login).
- Use a generic, centered spinner or pulsing icon.

## Task 3: Create Admin Loading State
- Create `src/app/admin/loading.tsx` to provide visual feedback when navigating within or into the admin panel.

## Task 4: Verify Suspense Boundaries
- Review `src/app/dashboard/page.tsx` and ensure any slow data fetching (e.g., fetching recent research history) is properly wrapped in a React `<Suspense>` boundary if it doesn't already use the top-level layout `loading.tsx`.
- If necessary, add `loading.tsx` to nested routes like `src/app/dashboard/history/loading.tsx`.
