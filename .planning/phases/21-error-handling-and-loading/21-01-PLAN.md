# Phase 21: Error Handling & Loading States - Plan 1: Global Error Handling & Error Boundaries

## Task 1: Create Root Error Boundary
- Create `src/app/error.tsx`.
- Implement a user-friendly error UI following the project's dark theme aesthetic.
- Include a "Try Again" button that calls `reset()`.
- Ensure it logs the error to the console (and Sentry, if applicable).

## Task 2: Create Dashboard Error Boundary
- Create `src/app/dashboard/error.tsx`.
- Implement an error state specifically for the dashboard layout.
- This ensures that if a specific dashboard feature fails, the sidebar and navigation remain intact, allowing the user to navigate away.

## Task 3: Create Admin Error Boundary
- Create `src/app/admin/error.tsx` to catch errors within the admin panel gracefully.
- Include a "Try Again" mechanism.

## Task 4: Verify Server Action/API Error Handling
- Review key API calls and server actions to ensure they throw meaningful errors or return `{ error: string }` objects that can be caught by the boundaries or displayed via UI toasts.
- Focus on the query submission and token ledger operations.
