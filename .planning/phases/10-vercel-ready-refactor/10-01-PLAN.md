<objective>
Eliminate the child-process + file-system architecture that causes the 5% stuck bug and blocks Vercel deployment. Convert all 4 data agents from CLI scripts (spawned via `execAsync`) into direct async function calls that return their data in memory.
</objective>

<context>
Current architecture:
```
route.ts → execAsync("npx tsx src/agents/search-agent.ts THYAO")
         → agent writes research/1234-thyao-kap.json
         → route.ts reads file with fs.readdirSync + fs.readFileSync
```

This causes:
1. **5% stuck** — progress updates at 5%, then child processes block until completion
2. **Vercel incompatible** — serverless functions have no persistent filesystem
3. **Race conditions** — multiple concurrent requests share the same `research/` directory
4. **Slow** — spawning 4 Node.js child processes adds ~8s overhead

Target architecture:
```
route.ts → const kapData = await runSearchAgent(ticker)
         → data returned directly as typed objects in memory
         → saved to Supabase columns
```
</context>
