<objective>
Replace the Puppeteer-based KAP scraper with direct HTTP API calls to KAP's internal endpoints. Puppeteer is slow (~10s per request), fragile (breaks when KAP changes DOM), and incompatible with Vercel serverless (no headless Chrome). KAP has internal JSON API endpoints that return structured data without a browser.
</objective>

<context>
Current KAP client (`src/lib/kap/client.ts`):
- Launches headless Chromium via `puppeteer-extra` with stealth plugin
- Navigates to `kap.org.tr`, types query into search box, scrapes DOM
- Returns whatever KAP shows â€” no company-level filtering
- Takes ~10-15s per request

KAP has internal API endpoints (used by their own frontend):
- `POST https://www.kap.org.tr/tr/api/memberDisclosureQuery` â€” search disclosures with filters
- `GET https://www.kap.org.tr/tr/api/member/{stockCode}` â€” company info by ticker
- `GET https://www.kap.org.tr/tr/api/disclosures` â€” recent disclosures with filters

These are undocumented but publicly accessible JSON endpoints. They don't require authentication â€” KAP's frontend calls them directly.
</context>
