<objective>
Fix news search relevance so that each query returns topic-specific news instead of generic Turkish business articles. Currently all queries get the same generic news because the extracted ticker (e.g., "ENERJI") is a meaningless search keyword.
</objective>

<context>
Current problem:
1. User types "Enerji Sektoru karsilastirmali raporu"
2. `extractTicker()` → "ENERJI" (first_word_fallback — not a real ticker)
3. `searchNews("ENERJI")` → Currents API returns generic Turkish news containing "enerji"
4. Same generic news appears for every query

Two-part fix needed:
A. **Ticker extractor** must differentiate between ticker queries and topic queries
B. **News agent** must use meaningful search keywords based on query type
</context>
