# Phase 18: Admin Dashboard Fixes & Enhancements
## Plan 18-02: Token Add/Remove — API Extension + EditUserDialog UI

**Wave:** 1
**Depends on:** —
**Files modified:**
- `src/app/api/admin/users/[userId]/route.ts`
- `src/components/admin/users/EditUserDialog.tsx`

### Objective
Enable admins to add or subtract tokens for any user directly from the Users management panel. Currently, the `EditUserDialog` only edits the user's role. The PUT API only accepts `role`. This plan adds a `token_adjustment` field to the PUT handler and a token management section to the dialog UI.

### Context & Rationale
Phase 11 introduced `token_ledger` for robust auditing. The correct approach for adjusting tokens is to insert into `token_ledger` (if available) and let a trigger update `users.tokens_balance`, OR directly update `users.tokens_balance` as fallback. We use a try/insert-to-ledger first, then direct-update fallback strategy in the API. The dialog shows the current `tokens_balance` and lets admin input a positive (add) or negative (subtract) adjustment amount.

<read_first>

- `src/lib/supabase/admin.ts` — admin client factory

<action>
In the PUT handler in `src/app/api/admin/users/[userId]/route.ts`:

1. Destructure `token_adjustment` (number | undefined) from `body` in addition to `role`
2. Handle token_adjustment when provided (non-zero):
   a. First, try inserting into `token_ledger`: `{ user_id: params.userId, delta: token_adjustment, reason: 'admin_adjustment', admin_id: adminId }`
   b. If `token_ledger` insert fails (table may not exist), fall back: fetch current `tokens_balance` from `users`, add `token_adjustment`, update `users.tokens_balance` directly — clamp to minimum 0
3. When `role` is undefined in body, skip the role update
4. When `token_adjustment` is undefined or 0, skip token logic
5. Audit log action: use `'ADJUST_USER_TOKENS'` with `details: { delta: token_adjustment }`
6. Return the updated user row (re-fetch after all updates)

Both `role` update and `token_adjustment` can be sent in the same request (the admin dialog sends both if changed). Handle each independently.
</action>

<acceptance_criteria>
- `PUT /api/admin/users/:id` with body `{ token_adjustment: 100 }` increases the user's token balance by 100
- `PUT /api/admin/users/:id` with body `{ token_adjustment: -50 }` decreases balance by 50 (min 0)
- `PUT /api/admin/users/:id` with body `{ role: 'admin' }` still works (no token change)
- `PUT /api/admin/users/:id` with both `role` and `token_adjustment` updates both
- Token adjustment is logged to `admin_audit_logs` with action `'ADJUST_USER_TOKENS'`
- Response is the updated user JSON
- When `token_adjustment` is 0 or undefined, no token writes occur
</acceptance_criteria>
</task>
