# Phase 18: Admin Dashboard Fixes & Enhancements
## Plan 18-01: Fix Analytics Charts (30-Day Zero-Fill)

**Wave:** 1
**Depends on:** —
**Files modified:**
- `src/app/api/admin/analytics/route.ts`

### Objective
Fix the "Günlük Kayıt Akışı" and "Araştırma Yoğunluğu" charts on the Admin Dashboard. Currently they show blank when there's no data in the last 30 days because the API only returns dates that have entries. Fix by generating the complete 30-day date range on the server and filling missing days with `count: 0`.

### Context & Rationale
The `groupByDate` helper in `analytics/route.ts` reduces raw rows into a sparse `{ date, count }` map. If the 7 sessions and 2 users were created before the 30-day window (or sparsely within it), the chart receives an empty array and renders "VERİ BULUNAMADI" or shows no bars. The `AnalyticsChart` component is otherwise correct — it uses `Math.max(heightPercentage, 4)` to ensure bars are visible, so fixing the data feed is sufficient.

<action>
Replace the `groupByDate` helper with a new `groupByDateFilled` function that:
1. Generates an array of the last N days (default 30) as ISO date strings (`YYYY-MM-DD`)
2. Counts each row's date from `item.created_at.split('T')[0]`
3. Returns the full array with `count: 0` for days with no data

Signature: `groupByDateFilled(data: {created_at: string}[], days: number = 30): {date: string; count: number}[]`

Replace both calls — `groupByDate(recentUsers || [])` and `groupByDate(recentSessions || [])` — with `groupByDateFilled(recentUsers || [])` and `groupByDateFilled(recentSessions || [])`.

Remove the old `groupByDate` function entirely.
</action>
