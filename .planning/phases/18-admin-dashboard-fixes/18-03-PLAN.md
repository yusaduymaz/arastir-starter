### Objective
Create the Sistem Logları admin page at `/admin/logs`, expose it in the AdminSidebar, and fix the admin dashboard quick-action button that currently points to `href: '#'`.

### Context & Rationale
In `src/app/admin/dashboard/page.tsx` line ~126, `{ label: 'Sistem Logları', href: '#' }` goes nowhere. The page needs to show meaningful log data. The best sources are:
1. `admin_audit_logs` — admin actions (role changes, token adjustments)
2. `agent_logs` — agent execution logs per research session (if the table exists; skip gracefully if not)

The page should be read-only — it displays logs but does not allow edits. Design matches the existing admin terminal aesthetic.

<read_first>

- `src/lib/supabase/admin.ts` — createAdminClient

<action>
Create `src/app/api/admin/logs/route.ts` with a GET handler:

1. Auth guard: `isAdmin()` check, return 403 if not admin
2. Parse query params: `limit` (default 50, max 200), `type` (optional: `'audit'` | `'agent'` | `''` for all)
3. Fetch `admin_audit_logs`: select `id, admin_id, action, target_type, target_id, details, created_at` ordered by `created_at DESC` with limit
4. Fetch `agent_logs`: try to select `id, session_id, agent_name, level, message, created_at` from `agent_logs` ordered by `created_at DESC` with limit — catch any error (table may not exist) and return `[]` instead
5. Merge both arrays, sort by `created_at DESC`, slice to `limit`
6. Return `NextResponse.json({ logs: mergedLogs, total: mergedLogs.length })`

Each log entry has a `source` field added: `'audit'` or `'agent'`.
</action>

<acceptance_criteria>
- `GET /api/admin/logs` returns `{ logs: [...], total: N }`
- Returns 403 for non-admin callers
- Each log entry has `source: 'audit'` or `source: 'agent'`
- `admin_audit_logs` entries always returned (or empty array on empty table)
- `agent_logs` fetch failure is caught gracefully — returns `[]`, does not crash the endpoint
- Logs sorted by `created_at` descending (newest first)
</acceptance_criteria>
</task>
