### Objective
Create the API Durumu (API Status) admin page at `/admin/api-status`, expose it in the AdminSidebar, and fix the admin dashboard quick-action button that currently points to `href: '#'`.

### Context & Rationale
In `src/app/admin/dashboard/page.tsx`, `{ label: 'API Durumu', href: '#' }` goes nowhere. The page should show the health / configuration status of all integrated RapidAPI providers and internal services (Supabase, Clerk). Because live pinging every provider on page load is expensive and rate-limited, we show env-key presence (configured vs missing) + Supabase connectivity as a fast alternative. We can optionally include a manual "Test" button per provider that does a single ping.

The provider list comes from the `data-sources/page.tsx` categories (15 providers) which are already documented in the project.

<action>
Create `src/app/api/admin/api-status/route.ts` with a GET handler:

1. Auth guard: `isAdmin()` check, return 403 if not admin

2. Check Supabase connectivity: try `createAdminClient().from('users').select('count', { count: 'exact', head: true })` — record `supabase: { status: 'ok' | 'error', latencyMs: number }`

3. Check env-key presence for RapidAPI providers. Map each provider to its env var name:
   - `RAPIDAPI_KEY` (shared key, check `process.env.RAPIDAPI_KEY`)
   - Individual provider keys: `RAPIDAPI_CNBC_KEY`, `RAPIDAPI_HAREM_KEY`, `RAPIDAPI_FINANCE_API_KEY`, `RAPIDAPI_EXCHANGE_RATES_KEY`, `RAPIDAPI_CRYPTO_NEWS_KEY`, `RAPIDAPI_YH_FINANCE_KEY`, `RAPIDAPI_YAHOO_RT_KEY`, `RAPIDAPI_CRYPTO_RSI_KEY`, `RAPIDAPI_FOREX_KEY`, `RAPIDAPI_TURKEY_FINANCIAL_KEY`, `RAPIDAPI_REALTIME_FINANCE_KEY`, `RAPIDAPI_TURKEY_NEWS_KEY`
   - For each: `configured: !!process.env[varName]`

4. Check Clerk: verify `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` and `CLERK_SECRET_KEY` are present

5. Return:
```json
{
  "services": {
    "supabase": { "status": "ok|error", "latencyMs": 42 },
    "clerk": { "configured": true }
  },
  "providers": [
    { "name": "KAP", "envVar": "RAPIDAPI_KEY", "configured": true },
    ...
  ],
  "checkedAt": "2026-05-19T12:00:00.000Z"
}
```
</action>

<acceptance_criteria>
- `GET /api/admin/api-status` returns 403 for non-admins
- Response contains `services.supabase.status` as `'ok'` or `'error'`
- Response contains `services.supabase.latencyMs` as a number
- Response contains `services.clerk.configured` boolean
- Response contains `providers` array with at least 12 entries
- Each provider entry has `name`, `envVar`, `configured` fields
- Response includes `checkedAt` ISO timestamp
- Supabase check failure is caught — returns `{ status: 'error', latencyMs: -1 }` instead of crashing
</acceptance_criteria>
</task>
