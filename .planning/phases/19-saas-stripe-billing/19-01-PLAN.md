## Pricing Tiers (to hard-code as constants)
| Tier      | Araştırma/ay | Fiyat       | Stripe Price ID Env Var               |
|-----------|-------------|-------------|---------------------------------------|
| free      | 5           | ₺0          | -                                     |
| starter   | 30          | ₺299/ay     | STRIPE_PRICE_STARTER_MONTHLY          |
| pro       | 100         | ₺799/ay     | STRIPE_PRICE_PRO_MONTHLY              |
| agency    | 500         | ₺2499/ay    | STRIPE_PRICE_AGENCY_MONTHLY           |
| enterprise| Özel        | Özel        | (contact sales — no Stripe price)     |

## Architecture Decision
- Use Stripe **Checkout** (redirect to Stripe hosted page) — simpler, PCI compliant, no embedded card form
- Use Stripe **Customer Portal** for subscription management — no custom cancel/upgrade UI needed
- Store `stripe_customer_id` and `stripe_subscription_id` on `users` table
- Add a `subscriptions` view/table for richer subscription state tracking
</context>

  <files>supabase/migrations/20260519000001_stripe_billing.sql</files>
  <action>
    Create the migration file with the following changes:

    1. Add columns to the `users` table:
       ```sql
       ALTER TABLE users
         ADD COLUMN IF NOT EXISTS stripe_customer_id TEXT UNIQUE,
         ADD COLUMN IF NOT EXISTS stripe_subscription_id TEXT UNIQUE,
         ADD COLUMN IF NOT EXISTS subscription_status TEXT NOT NULL DEFAULT 'inactive'
           CHECK (subscription_status IN ('active', 'trialing', 'past_due', 'canceled', 'inactive')),
         ADD COLUMN IF NOT EXISTS current_period_end TIMESTAMP WITH TIME ZONE,
         ADD COLUMN IF NOT EXISTS cancel_at_period_end BOOLEAN NOT NULL DEFAULT false;
       ```

    2. Create index for Stripe customer lookup:
       ```sql
       CREATE INDEX IF NOT EXISTS idx_users_stripe_customer ON users(stripe_customer_id);
       ```

    3. Update the `tier` check constraint to include 'pro' and 'enterprise':
       ```sql
       ALTER TABLE users DROP CONSTRAINT IF EXISTS users_tier_check;
       ALTER TABLE users ADD CONSTRAINT users_tier_check
         CHECK (tier IN ('free', 'starter', 'pro', 'agency', 'enterprise'));
       ```

    4. Set default monthly token grants per tier (comment-documented):
       ```
       -- free: 5 tokens/month (manual grant or webhook)
       -- starter: 30 tokens/month
       -- pro: 100 tokens/month
       -- agency: 500 tokens/month
       -- enterprise: custom (admin-managed)
       ```
